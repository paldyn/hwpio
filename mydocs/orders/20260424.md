## 1. Task #265 — HWP 3.0 파일 감지 + 친절한 에러 메시지

### 배경

@jangster77 님의 제보 (2026-04-23). `samples/issue_265.hwp` 를 rhwp 의 어떤 경로에서도 열지 못함. 원인: **HWP 3.0** 바이너리 포맷 (1996~1998년 경 한글 3.0). 현재 rhwp 는 HWP 5.0 + HWPX 만 지원.

### 처리 방침
작업지시자 지시: "HWP 3.0 정식 지원은 이번 마일스톤 범위 아님. 감지 + 친절한 에러 메시지까지만".

### 변경 범위
- **코드** `src/parser/mod.rs`:
  - `FileFormat::Hwp3` variant + `detect_format` 에 17바이트 프리픽스 체크
  - `ParseError::UnsupportedFormat { format, hint }` + `Display` 확장
  - `parse_document` 에서 Hwp3 → 한글 안내 메시지 반환
- **코드** `src/error.rs` — **발견된 추가 버그 수정**:
  - `From<ParseError> for HwpError` 등 3개 `From` 이 `{:?}` (Debug) 를 쓰고 있어 친절한 Display 메시지가 사용자에게 도달하지 못함. `{e}` (Display) 로 전환. CFB 매직 바이트 내부 누출도 함께 해결
- **코드** `rhwp-studio/src/main.ts` — **UI 전달 누락 수정**:
  - 기존: 상태 표시줄 (22px, `overflow: hidden`) 에만 쓰고 있어 긴 메시지 대부분 가려짐
  - 수정: `showLoadError` 유틸 + 우상단 토스트 (`showToast` 기존 유틸 재사용). 3 로드 경로 (`loadFile` · `loadFromUrlParam` · `open-document-bytes` 이벤트) 통합
  - `open-document-bytes` 이벤트 핸들러가 WASM throw 를 catch 하지 않아 "파일 열기" 커맨드 에선 에러가 완전 묵살되던 문제도 함께 수정
- **테스트** 6건 (detect + parse + Display 회귀 방어)
- **WASM 재빌드**: 4,076,166 → 4,081,046 bytes (+4,880 bytes)

## 2. Task #280 — 수식 SVG 렌더링 폰트 스택 재정렬

### 배경

작업지시자 제보: `samples/equation-lim.hwp` (`lim _{h→0} {f(2+h)-f(2)} over h`) 가 한컴 PDF 대비 "폰트가 볼드, 크기가 다름". 5단계로 수행.

### 조사 (이 타스크의 실질적 가치)

"크기가 다르다" 는 인상이 PDF 콘텐츠 스트림(`BT /F1 110 Tf ... lim ... BT /F1 92 Tf ... f ...`)에서 `lim=110pt` vs 본문=92pt 로 보여 "함수명 1.2x 확대 규칙" 의심으로 출발. 그러나:

- `exam_math.hwp` 내 16개 수식의 bbox 높이 비교 → `"b= log 2"` = 1125 HWPUNIT (font_size * 1.02) 로 단순 수식과 동일
- **함수명 확대 규칙 없음** 을 데이터로 확정
- PDF 의 110/92 차이는 HyhwpEQ 폰트의 ASCII 글리프(l,i,m) vs PUA 수식 글리프(f,h,(,)) em 박스 차이를 HWP 엔진이 보정한 것

이 판단 오류를 피한 것이 이번 타스크의 핵심 — 1.2x 배율을 잘못 적용했으면 오히려 다른 수식들이 깨졌을 것.

### 변경 범위

- **코드** `src/renderer/equation/svg_render.rs` + `canvas_render.rs` 각 1줄:
  - `EQ_FONT_FAMILY` / `set_font` 에서 **`Cambria Math` 제거** (Windows 볼드 인상의 근본 원인)
  - `'Pretendard'` 제거 (산세리프, 수식 부적합)
  - `'STIX Two Text'`, `'Times New Roman'`, `'Times'` 추가 (클래식 세리프 폴백 체인)
  - 최종 스택: `'Latin Modern Math', 'STIX Two Text', 'STIX Two Math', 'Times New Roman', 'Times', serif`
  - `Latin Modern Math` 첫 번째 유지 → `svg.rs:332` 의 폰트 임베딩 파이프라인 호환
- **샘플** `samples/equation-lim.{hwp,pdf}` 커밋 (재현용)
