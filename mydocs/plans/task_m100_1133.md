# Task M100-1133 — 중첩 표 셀 세로 정렬/높이 산정 보정

## 배경

GitHub Issue #1133은 표 셀 내부에 중첩된 1x1 표가 있는 문서에서 다음 문제가 발생하는 사례이다.

- 중첩 표가 보이지만 선택/편집 대상에서 안정적으로 잡히지 않음
- 외부 셀/행 높이 계산에 중첩 표의 실제 콘텐츠 높이가 충분히 반영되지 않아 아래 내용과 겹침

작업지시자가 제공한 재현 샘플:

- `samples/hwpx/issue_1133.hwpx`
- `samples/issue_1133.hwp`
- `pdf-large/hwpx/issue_1133.pdf`

## 초기 판정

페이지 2의 `지원서 접수` 영역을 덤프한 결과, 문제 셀은 외부 표 `pi=29`의 `cell[3]`이다.

- 외부 셀: `r=1,c=1`, `valign=Center`
- 마지막 문단 `p[5]`에 내부 1x1 표가 존재
- `p[5]`의 line segment는 `vpos=8552`, `lh=1400`만 가진다
- 내부 표의 저장 cell height는 `282 HU`로 매우 작지만, 실제 내부 문단은 여러 줄로 구성됨

따라서 핵심 가설은 다음과 같다.

> 중첩 표가 있는 문단의 실제 끝 위치를 `line_seg.vpos + line_seg.line_height`로만 계산해서, `line_seg.vpos + nested_table_actual_height`가 행 높이/세로 가운데 정렬 계산에 반영되지 않는다.

## 구현 계획

1. 중첩 표 포함 문단의 실제 끝 위치 계산을 보강한다.
   - 문단 line height와 nested table 실제 높이 중 큰 값을 문단 높이로 사용
   - 전체 셀 콘텐츠 높이는 각 문단의 `vpos + 보정 문단 높이` 최댓값으로 산정
2. 행 높이 측정 경로와 렌더링 세로 정렬 경로가 같은 의미의 콘텐츠 높이를 사용하도록 맞춘다.
   - `height_measurer`
   - `table_layout`
   - `table_partial`
3. HWP/HWPX 샘플 모두에서 page 2 SVG를 재생성하여 작업지시자 시각 판정을 받는다.
4. 성공 시 WASM 빌드와 회귀 테스트를 진행한다.
