# Task #604 — Document IR 표준 정합화 + HWP3 wrap zone cs/sw 인코딩 정정

## 1. 본질

### 1.1 결함 (Issue #604)

`hwp3-sample5.hwp` page 4 (HWP3 native): pi=74 그림 (User level programs/Kernel/Hardware 다이어그램, 126.4×94.5mm, Square wrap, tac=false) 의 우측에 wrap text (pi=75, "커널의 가장 밑바탕은...") 가 정상 배치되지 않고 **그림 좌측 + 그림 위 + 그림 아래** 에 산재.

### 1.2 데이터 검증 (PR #589 머지 후)

```
--- 문단 0.74 --- cc=1, text_len=1, controls=1 [쪽나누기]
  텍스트: "￼"
  ls[0]: ts=0, vpos=0, lh=1440, ..., cs=35460, sw=15564
  [0] 그림: 126.4×94.5mm, tac=false, Square wrap

--- 문단 0.75 --- cc=415, text_len=415
  텍스트: "커널의 가장 밑바탕은..."
  ls[0]: ts=0,   vpos=0, ..., cs=0,     sw=0       ← ❌ wrap zone 미설정
  ls[1]: ts=17,  vpos=0, ..., cs=0,     sw=0       ← ❌
  ls[2]: ts=33,  vpos=0, ..., cs=0,     sw=0       ← ❌
  ls[3]: ts=50,  vpos=0, ..., cs=35460, sw=15564   ← ✓
  ls[4~]:                     cs=35460, sw=15564   ← ✓
```

### 1.3 본질 진단

**근본 원인 1 — HWP3 파서 (`src/parser/hwp3/mod.rs:1399-1407`)**:
```rust
let line_cs_sw = current_zone.and_then(|(cs, sw, pgy_start, pgy_end)| {
    if pic_wrap_zone.is_some() || (linfo.pgy >= pgy_start && linfo.pgy < pgy_end) {
        Some((cs, sw))
    } else {
        None
    }
});
```
pi=75 의 첫 3 줄 `linfo.pgy` 가 pi=74 그림의 wrap zone pgy 범위 (`pgy_start..pgy_end`) **밖** 으로 판정되어 cs/sw=0 설정. → wrap zone 인코딩 결함.

**근본 원인 2 — Document IR 표준 부재**:
- `LineSeg.column_start/segment_width` 의 의미가 IR 표준에 명시 안 됨 (단위/원점/0 의미 모호)
- `Paragraph.wrap_precomputed` 플래그가 HWP3 휴리스틱을 IR 에 누설 (PR #589 보완6 도입, 본 task 정정 대상)
- HWP5/HWPX 파서는 wrap_precomputed 미설정 → HWP5/HWPX wrap text 도 동일 결함 잠재

(분석 자료: `mydocs/tech/document_ir_parser_relationship_analysis.md` 16KB, `mydocs/tech/hwp5_wrap_precomputed_analysis.md`, `mydocs/tech/document_ir_wrap_zone_standard_review.md`)
