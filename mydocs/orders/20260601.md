## Task #18 (HanPage) — Windows 실행 파일명 `rhwp-desktop.exe` → `HanPage.exe` (mainBinaryName 누락)

- **이슈**: [paldyn/HanPage#18](https://github.com/paldyn/HanPage/issues/18) (M100) · **브랜치**: `local/task18` (origin/main 기준)
- **상태**: 이슈 등록 + 분기 + 계획서 작성 → **승인 대기**
- **증상**: Windows 설치 시 본체 실행 파일이 `rhwp-desktop.exe`로 노출(설치 마법사 파일명 `HanPage_0.7.13_x64-setup.exe`는 정상). 설치되는 .exe·시작메뉴 바로가기 대상·작업관리자 프로세스명·"실행 중 앱 종료" 프롬프트가 모두 `rhwp-desktop.exe`.
- **원인(확정)**: Tauri v2는 `mainBinaryName` 미지정 시 메인 바이너리명을 Cargo `[package] name`(=`rhwp-desktop`)에서 가져옴(v1의 productName 자동 리네임과 다름). `tauri.conf.json`에 `mainBinaryName` 없음. → #13에서 "크레이트명 비노출" 판단이 **Windows exe명에 한해 어긋남**.
- **수정 방침**: `tauri.conf.json` 최상위 `"mainBinaryName": "HanPage"` **1줄 추가**. Rust 크레이트 식별자(`rhwp-desktop`/`rhwp_desktop_lib`)는 **유지**(mainBinaryName이 "노출 exe명"과 "크레이트명"을 분리). 실제 반영엔 새 릴리스 태그 `hanpage-desktop-v0.7.14` 발행 필요.
