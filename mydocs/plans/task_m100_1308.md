# Task #1308 수행 계획서

## 이슈

- GitHub Issue: #1308
- 제목: Shift+Enter 강제 줄바꿈 후속 줄에 문단 들여쓰기/내어쓰기 미적용

## 배경

rhwp-studio에서 `Shift+Enter`를 누르면 `Enter`처럼 문단을 새로 만들지 않고, 같은 문단 안에
강제 줄바꿈 문자(`\n`)를 삽입한다.

한컴에디터 기준으로 강제 줄바꿈 뒤 텍스트는 같은 문단의 다음 visual line이다. 따라서 문단의
들여쓰기/내어쓰기, 정렬, 커서 이동은 새 문단이 아니라 같은 문단의 후속 줄 규칙을 따라야 한다.

현재 rhwp는 일부 경로에서 단일 `LINE_SEG` 범위 안의 중간 `\n`을 별도 visual line으로 분리하지 않아,
강제 줄바꿈 뒤 줄이 문단 후속 줄로 배치되지 않는 문제가 있다.

## 재현 조건

- 같은 문단 내부 텍스트에 `\n`이 포함됨
- 단일 `LINE_SEG`가 문단 전체 범위를 가짐
- `\n`이 줄 끝이 아니라 중간에 위치함
- 문단에 들여쓰기/내어쓰기 속성이 있음

## 테스트 샘플

- `samples/eq-002.hwp`
- `samples/hwpx/eq-002.hwpx`
- `pdf-large/hwpx/eq-002.pdf`

예:

```text
가나\n다라
```

기대:

- 1번째 visual line: `가나`, `has_line_break=true`
- 2번째 visual line: `다라`, `char_start`는 `\n` 다음 문자 위치
- 렌더러는 2번째 줄을 같은 문단의 후속 줄로 취급해 문단 들여쓰기/내어쓰기 산식을 적용

## 원인

`src/renderer/composer.rs`의 `compose_lines()` 일반 경로는 기존에 다음 방식이었다.

- `line_text.ends_with('\n')`만 강제 줄바꿈으로 표시
- 중간 `\n`은 별도 `ComposedLine`으로 분리하지 않음

이 때문에 단일 `LINE_SEG` 안에 `가나\n다라`가 들어오면 composer가 한 줄로 유지하고,
renderer는 `다라`를 후속 줄로 판단할 수 없다.

## 구현 계획

1. `compose_lines()` 일반 경로에서 `line_text`를 문자 단위로 순회한다.
2. 중간 `\n`을 만나면 현재 조각을 `ComposedLine`으로 push하고 `has_line_break=true`를 지정한다.
3. `\n` 다음 문자부터 새 `ComposedLine`을 생성한다.
4. 각 조각의 `char_start`는 원문 paragraph char index를 유지한다.
5. 끝에 위치한 `\n`은 빈 후속 줄을 중복 생성하지 않는다.
6. 기존 TAC 특수 경로는 유지한다.

## 완료 기준

1. 단일 `LINE_SEG` 안의 중간 `\n`이 별도 `ComposedLine`으로 분리된다.
2. `\n` 앞 visual line만 `has_line_break=true`가 된다.
3. `\n` 뒤 visual line의 `char_start`가 `\n` 다음 문자 위치를 가리킨다.
4. 끝의 `\n`은 빈 후속 줄을 중복 생성하지 않는다.
5. 기존 composer 테스트와 전체 lib 테스트가 통과한다.
