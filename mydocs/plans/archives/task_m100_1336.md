# 수행계획서 — 페이지네이션 body 초과 줄 배치 (M100 #1336)

이슈: edwardkim/rhwp#1336 (선행 #1335 조사 분리)

## 1. 배경

`rhwp export-svg` 시 일부 페이지에서 콘텐츠가 body 영역 하단(1092.3px)을 넘어 배치된다
(`LAYOUT_OVERFLOW`). #1335 Stage 1 에서 단 구분선 길이에 전파되는 별개 버그로 분리됨.

## 2. 조사 결과 — 두 현상 분리

대상 문서 `samples/3-09월_교육_통합_2024-구분선아래20구분선위20.hwp`, 정답 = 한글 2022 PDF.

### (A) 미주(endnote) 다단 영역 대량 초과 — **명확한 버그**

- p22(global_idx=21)의 미주 2단 영역에서 **단 0(35 items)** 이 body 를 크게 초과.
  `LAYOUT_OVERFLOW` 경고: para 1156~1159 가 y=1102~1174 까지 배치(최대 +82px).
- typeset 누적(`used=1010.7px`)과 실제 layout 렌더 위치(~1174px)가 **~160px 불일치**.
  → 미주 다단 컬럼의 typeset 높이 산정과 layout 배치 높이가 어긋나, 단 0 에 항목이
    과적되고 다음 컬럼/페이지로 넘기지 못함.
- 메모리 경고 영역: `tech_endnote_tail_backtrack_atomic_vs_text`,
  `tech_trailing_model_no_ssot` (미주 trailing 전면 통일 금지).

### (B) 본문 마지막 줄 ~10px 드리프트 — **노이즈 바닥(범위 제외 권장)**

- p6·9·11·17·18·19: 본문 max TextLine 이 body 를 7~18px 초과. 그러나 PDF 최하단
  텍스트는 전 페이지 일관 1095.1px 이고 rhwp 는 그보다 7~18px 아래.
- "1줄 과배치"가 아니라 **줄간격 누적 드리프트(~0.2px/줄 × 수십 줄)**. `LAYOUT_OVERFLOW`
  경고도 안 뜬다(임계 미만).
- 줄높이/줄간격 누적 모델 수정 필요 → 고위험·저효익(노이즈 바닥). #1335 와 동류.

## 3. 권고 스코프

**#1336 은 (A) 미주 다단 영역 대량 초과로 한정.** (B) 본문 드리프트는 별개 노이즈
바닥 이슈로 본 타스크 제외(필요 시 별도 등록/보류).

## 4. 접근 방향 (가설)

미주 다단 컬럼 채움 시, typeset 의 항목 높이 누적과 layout 의 실제 배치 높이를 일치
시켜 단 0 과적을 차단:
- 미주 다단 컬럼의 fit 판정 높이 ↔ layout 렌더 높이 정합(본문 다단 `height_for_fit`
  분기와 유사한 미주 경로 점검: `typeset.rs` 미주 흐름 + `layout/picture_footnote.rs` /
  미주 area 배치).
- 단 0 가 available 초과 시 잔여 항목을 단 1 또는 다음 페이지로 이월.

## 5. 위험

- 미주 영역은 회귀 매우 민감(메모리 2건 경고). 미주 trailing 전면 통일 금지.
- 다단 미주 보유 샘플 회귀 점검 필수(exam_*, hwpspec 등).

## 6. 승인 요청

- 동의 시 구현계획서(Stage 1 근본원인 확정 → Stage 2 수정 → Stage 3 검증) 작성.
