## 3. 기술 전략: Rust 네이티브

**WASM(WebView)이 아닌 Rust → iOS 네이티브 빌드**

| | 한컴독스 (WebView 하이브리드) | 알한글 (Rust 네이티브) |
|---|---|---|
| 렌더링 | 웹 엔진 | **Metal / Core Graphics** |
| 성능 | 브라우저 오버헤드 | **네이티브 최적화** |
| 파일 연동 | 제한적 | **iOS Files 앱 직접 통합** |
| 메모리 | JS GC | **Rust 직접 관리** |
| 스크롤 | 웹 스크롤 | **UIKit 네이티브 60fps** |
| 입력 | 키보드 문제 多 | **네이티브 키보드 + Pencil** |

빌드: `cargo build --target aarch64-apple-ios` + Swift UI FFI

## 4. 핵심 기능: 레이어 구조

### 파일 구조

```
┌─────────────────────────────┐
│  어노테이션 레이어           │  ← Apple Pencil 필기, 형광펜, 메모
├─────────────────────────────┤
│  편집 가능한 문서            │  ← 텍스트 수정, 표 편집
├─────────────────────────────┤
│  원본 HWP                   │
└─────────────────────────────┘
```

### .rhwp 파일 포맷

HWP 원본 + 어노테이션 레이어를 **하나의 파일**로 묶음.

### 내보내기

| 내보내기 | 대상 | 내용 |
|---------|------|------|
| .rhwp | 같은 앱 사용자 | HWP + 어노테이션 전체 보존 |
| .hwp | 선생님 (한컴 사용) | 텍스트 편집만 반영 |
| .pdf | 누구나 | HWP + 어노테이션 합쳐서 평탄화 |

## 5. 사용 시나리오

### 학생: 시험 문제 풀기

```
카카오톡에서 시험문제.hwp 받음
  → 탭 한 번으로 열기
  → 답안은 타이핑 (키보드)
  → 풀이 과정은 Apple Pencil로 필기
  → 저장 (.rhwp)
  → 선생님에게 .pdf로 전송
  → 나중에 다시 열어서 복습
```

### 선생님: 채점

```
학생이 보낸 시험문제.rhwp 열기
  → 학생 답안 (타이핑) + 풀이 과정 (필기) 확인
  → 빨간 펜으로 채점 + 코멘트
  → 학생에게 다시 전송
```

### 레이어 누적

| 레이어 | 작성자 | 색상 |
|--------|--------|------|
| 원본 시험 문제 | 선생님 | (문서) |
| 답안 타이핑 | 학생 | (문서 편집) |
| 풀이 과정 필기 | 학생 | 파란펜 |
| 채점 + 코멘트 | 선생님 | 빨간펜 |
| 오답 정리 | 학생 | 초록펜 |

## 9. 고객의 목소리 (VOC)

### 클리앙 2022: "아이패드에서 한글파일을 열어서 필기 할수 있는 어플이 있을까요?"

- 학원 족보(HWP)를 iPad에서 풀고 싶은 학부모
- GoodNotes에서 HWP가 안 열림
- 유일한 해결책: 한컴뷰어 → 인쇄 → PDF → GoodNotes (4단계)
- **2022년 질문, 2026년 현재까지 답 없음**

### GoodNotes/Notability 한계

- 두 앱 모두 **플러그인 API 없음** — 서드파티 파일 포맷 지원 불가
- HWP를 PDF로 변환하는 것이 유일한 방법
- 변환 과정에서 표 깨짐, 서식 손실

### 알한글이 주는 답

```
기존: HWP 받음 → PC 한컴 → PDF 변환 → iPad 전송 → GoodNotes → 필기 (5단계)
알한글: HWP 받음 → 탭 → 필기 (2단계)
```

## 11. SVG 벡터 + 렌더 트리의 교육 특화 가치

### 기술적 우위 (ThorVG POC에서 검증됨)

렌더 트리(JSON) 추상화로 렌더러 교체가 자유로움:
- 웹: Canvas 2D
- iPad: Core Graphics / Metal
- 내보내기: ThorVG Native / PDF

### 교육 시나리오에서의 가치

| SVG 벡터 특성 | 학습 가치 |
|--------------|----------|
| 무한 줌 선명도 | 수식 확대해서 풀기, 작은 표 셀 확대 |
| 요소별 개별 접근 | 문제 번호 탭 → 해당 문제로 이동 |
| 텍스트 검색 | "삼각함수" 검색 → 관련 문제 하이라이트 |
| 셀 단위 터치 선택 | 표의 특정 셀 탭 → 답안 입력 |
| 어노테이션 좌표 정합 | Pencil 필기가 문서 요소와 정확히 매칭 |
| 경량 메모리 | 페이지당 수 KB — iPad 메모리 효율적 |

### 그림(PDF) vs 조판(알한글)

**PDF는 그림이고, 알한글은 조판이다.**

| | PDF (그림) | 알한글 (조판) |
|---|---|---|
| 본질 | 인쇄물의 사진 | 살아있는 문서 |
| 편집 | 그림 위에 덧칠 | 문서를 수정 |
| 검색 | 될 수도 안 될 수도 | 항상 가능 |
| 확대 | 흐려짐 | 선명 |
| AI 이해 | 그림을 읽어야 함 (토큰 수만 개) | 데이터로 접근 (토큰 수백 개) |
| 자동 채점 | OCR 의존 (부정확, 고비용) | 구조 데이터 (정확, 저비용) |

## 12. AI 학습 분석 플랫폼

### 데이터 흐름

```
알한글 레이어 데이터:
  ├─ 원본 문서 구조 (문제 번호, 배점, 유형)
  ├─ 학생 답안 (타이핑 텍스트)
  ├─ 학생 필기 (풀이 과정, 스트로크 타임스탬프)
  ├─ 선생님 채점 (○×△, 점수, 코멘트)
  └─ 시간 데이터 (문제별 풀이 시간)
       ↓
     AI 분석
       ↓
  "3번 삼각함수 문제에서 sin→cos 치환을 반복 틀림"
  "풀이 시간이 평균 2배 — 개념 이해 부족 추정"
  "유사 유형 5문제 추천"
```

### 자동 채점 워크플로우

```
학생 30명 .rhwp 제출
  → AI 자동 채점 (1차): 객관식 자동 ○×, 서술형 예상 점수
  → 선생님 검토 (2차): 승인 또는 수정, 코멘트 추가
  → 학생에게 반환
  → 2시간 → 20분
```

### 토큰 비용 비교

| | PDF → AI | 알한글 → AI |
|---|---|---|
| 입력 | 이미지 (수 MB) | 구조화 JSON (수 KB) |
| 토큰 | 수만 개 | 수백 개 |
| 학생 1명당 | 수백 원 | 수 원 |
| 30명 한 반 | 수만 원 | 수백 원 |
