# 수행 계획서 — Task M100-823: headless macOS PNG 렌더 CoreText fontd hang 방지

- 이슈: https://github.com/edwardkim/rhwp/issues/823
- 제목: PNG 렌더가 headless macOS에서 미설치 폰트 fallback 시 hang
- 작성일: 2026-06-04
- 브랜치: `local/task_m100_823`

## 1. 문제 요약

`SkiaLayerRenderer::render_raster_with_options` 기반 PNG 렌더가 headless macOS
(GitHub Actions `macos-latest` 등)에서 무한 대기할 수 있다. HWP 본문이 지정한
`한컴*`, `새*`, `함초롬*` 계열 폰트가 시스템에 없을 때 Skia/CoreText
`match_family_style` 호출이 downloadable font lookup을 트리거하고, headless
환경에서 `fontd` 응답자가 없어 `mach_msg2_trap`에서 영구 차단되는 경로다.

SVG export와 interactive desktop은 주 영향권 밖이며, 실질 영향은 CI/빌드팜의
PNG raster path에 집중된다.

## 2. 목표

시스템에 없는 font family에 대해서는 CoreText `match_family_style` 호출 자체를
하지 않도록 사전 필터링한다.

핵심 원칙:

- `FontMgr::family_names()` 결과를 `SkiaLayerRenderer` 생성 시 1회 캐시한다.
- fallback chain에서 custom typeface는 기존처럼 우선 사용한다.
- 시스템 fallback은 family가 캐시에 있을 때만 `match_family_style`을 호출한다.
- 마지막 `legacy_make_typeface(None, style)` fallback은 유지한다.

## 3. 현행 영향 파일

현재 `devel` 기준 확인된 호출부:

| 파일 | 호출부 | 조치 |
|------|--------|------|
| `src/renderer/skia/renderer.rs` | `SkiaLayerRenderer` 생성자 | 시스템 family `HashSet` 캐시 추가 |
| `src/renderer/skia/text_replay.rs` | 본문 fallback chain | missing system family skip |
| `src/renderer/skia/text_replay.rs` | `make_mark_font`의 `DejaVu Sans` 단발 호출 | 캐시에 있을 때만 호출 |
| `src/renderer/skia/renderer.rs` | `make_form_font` CJK fallback chain | missing system family skip |
| `src/renderer/skia/equation_conv.rs` | 수식 폰트 fallback chain | 캐시 전달 또는 helper 적용 |

## 4. 범위

포함:

- Skia PNG/raster path의 시스템 폰트 fallback 안전화.
- 단위 테스트 또는 macOS 외 환경에서도 결정적으로 검증 가능한 helper 테스트.
- 기존 SVG path 동작 보존.
- custom font path(`with_font_paths`) 동작 보존.

제외:

- CoreText/fontd 자체 설정 변경.
- 폰트 설치/CI 이미지 변경.
- SVG 수식 렌더러 또는 web canvas 렌더러 변경.

## 5. 검증 계획

필수:

- `cargo fmt --check`
- `cargo test --release --lib`
- Skia 관련 신규/수정 테스트
- 가능하면 `cargo test --release --test ...` 범위 테스트

권장:

로컬 테스트는 "missing family에는 `match_family_style`을 호출하지 않는 구조"를
가드하고, 가능하면 maintainer 환경에서 실 PNG 렌더로 보완한다.
