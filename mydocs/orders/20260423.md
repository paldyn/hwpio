## 10. PR #256 — Task #146 텍스트 정렬 SVG↔PDF 일치 (by @planet6897)

v0.2.1 사이클 OLE/Chart/EMF 기여자 @planet6897 의 두 번째 대형 기여. text-align.hwp 1페이지가 한컴 PDF 출력과 시각적으로 달랐던 문제를 **좌표 정밀 비교로 원인 규명** 후 3건 수정.

### 진단 방법
`mutool draw -F stext` 로 PDF 문자 좌표 추출 → SVG 좌표와 1:1 비교 → 수치로 원인 특정. 이슈 #253 (한컴 PDF Visual Diff 하네스 구상) 의 실제 사례가 됨.

### 코드 수정 3건
1. **Geometric Shapes 전각 처리** (`text_measurement.rs`)
   `is_fullwidth_symbol` 범위에 U+25A0-U+25FF (□■▲◆○) 추가. "□" 이 반각으로 측정되어 후속 글자가 좌측 붕괴하던 문제 해결. 제목 x 좌표 PDF 환산과 0.01 px 일치.
2. **TAC 표 선행 텍스트 폭 반영** (`layout.rs`)
   pagination 이 TAC 문단에 `PageItem::Table` 만 발행할 때 선행 공백이 표 x 좌표에 미반영되던 문제 해결 + block 취급 TAC fallback.
3. **Heavy display face → visual bold** (`style_resolver.rs` · `mod.rs` · `svg.rs`)
   HY헤드라인M / HY견고딕 등 face 자체가 굵은 display 폰트가 regular 로 fallback 되던 문제 해결.

### 기각된 초기 가설 (디버깅 지식 자산)

- 표 셀 `1,000항목` → `1, 000항목` (콤마 뒤 과다 공백)
