# Task #703 수행 계획서

**제목**: BehindText/InFrontOfText 표가 paragraph 높이에 포함되어 trailing 항목이 다음 페이지로 밀림
**Issue**: #703
**브랜치**: `local/task703`
**작성일**: 2026-05-08
**마일스톤**: M100 (v1.0.0)

## 1. 문제 정의

### 증상

| 파일 | SVG | 한글2022 PDF | 비율 |
|------|-----|--------------|------|
| `samples/basic/calendar_year.hwp` | 2 | 1 | 2.00× |
| `samples/통합재정통계(2010.11월).hwp` | 2 | 1 | 2.00× |
| `samples/통합재정통계(2011.10월).hwp` | 2 | 1 | 2.00× |

세 파일 모두 마지막 작은 문단(빈 단락 또는 PushButton)이 page 2 로 밀려나 page 2 가 거의 비어있음.

### 정량 분석 (디버그 인스트루먼트 결과)

| 파일 | trailing pi | renderer cur_h | available | remaining | first_line_h | fit 부족 |
|------|-------------|----------------|-----------|-----------|--------------|----------|
| calendar_year | 12 (PushButton) | 984.08 | 991.57 | 7.49 | 26.45 | 18.96 px |
| 통합재정통계(2010.11월) | 14 (빈) | 751.03 | 766.19 | 15.16 | 16.00 | 0.84 px |

`typeset.rs:1107-1112` 의 fit 검사가 트리거점:

```rust
if (st.current_height >= available || remaining < first_line_h)
    && !st.current_items.is_empty() {
    st.advance_column_or_new_page();
}
```

### 누적 height drift

| 파일 | HWP 기록 (hwp_used) | renderer cur_h | drift |
|------|--------------------|----------------|-------|
| calendar_year (after pi=11) | 952.3 | 984.1 | **+31.8 px** |
| 통합재정통계 (after pi=13) | 745.1 | 751.0 | **+5.9 px** |

calendar_year 의 drift 중 23.77 px 는 pi=1 (글뒤로 1×1 wrapper Table 을 가진 빈 paragraph) 단일 문단에서 발생:

- HWP 기여: `lh + ls = 1600 HU = 21.33 px` (라인 한 줄)
- 렌더러 기여: **45.10 px** (= 21.33 + 23.77 excess)

## 2. Root cause

`src/renderer/height_measurer.rs` 에 BehindText / InFrontOfText 표(/도형) 분기 누락.

```bash
$ grep -n 'BehindText\|InFrontOfText' src/renderer/height_measurer.rs
# (결과 없음)
```

`pagination/engine.rs:976` 와 `typeset.rs` 의 `process_controls` 는 BehindText 표를 본문 흐름에서 제외하나, **paragraph 측정 단계에서는 캐리어 paragraph 의 measured height 에 표 높이가 포함**되어 cur_h 누적 시 잘못 가산됨.

## 3. 정정 방향

### 옵션 A — height_measurer 본질 정정 (권장)

paragraph 측정 시 컨트롤 순회 분기에 다음 추가:

```rust
// BehindText/InFrontOfText 표·도형은 본문 흐름 외 — 높이 가산 제외
if matches!(common.text_wrap, TextWrap::BehindText | TextWrap::InFrontOfText) {
    continue;
}
```

**영향**:
- calendar_year 의 pi=1 drift (23.77 px) 직접 해소
- 통합재정통계 의 drift 일부 해소 (한 페이지에서 글뒤로 표 사용시)
- TopAndBottom/Square 등 다른 wrap 은 영향 없음

**회귀 위험**: 글뒤로/글앞으로 표를 워터마크/배경 box 로 사용하는 다른 문서. 196 샘플 회귀 검증으로 확인.

### 옵션 B — fit 검사 tolerance 보강 (보조)

`typeset.rs:1107` 의 `remaining < first_line_h` 조건에 0.5 px epsilon 추가:

```rust
if (st.current_height >= available + 0.5
    || remaining + 0.5 < first_line_h)
    && !st.current_items.is_empty()
```

**영향**: 통합재정통계 의 0.84 px borderline 케이스 단독 해소.
**회귀 위험**: 낮음 (이미 다른 곳에 0.5 px epsilon 사용 중).

### 본 task 선택: **옵션 A 단독**

- A 가 본질 정정. 통합재정통계의 0.84 px 부족도 글뒤로/글앞으로 표가 본 문서에 있으면 자동 해소될 가능성 (분석 단계에서 확인 예정).
- 만약 통합재정통계가 A 적용 후에도 회귀 없이 잔존하면, 그 시점에 옵션 B 를 별도 task 로 등록 (본 task 범위 외).
