# Task #1261 작업 계획

## 문제

`samples/3-10월_교육_통합_2022.hwp`를 rhwp-studio에서 확인할 때 5쪽 `문28)` 조건 박스 내부의 번호, 수식, 본문 줄이 한컴오피스 기준과 다르게 서로 겹쳐 보인다.

작업지시자 첨부 화면 기준으로 한컴오피스에서는 조건 박스 내부 문장이 줄 간격을 유지하지만, rhwp-studio에서는 첫 조건 줄 주변에서 원문자, 수식, 본문 텍스트가 같은 높이에 압축되어 표시된다.

## 목표

1. `문28)` 조건 박스 내부 겹침의 원인이 문단 줄높이, 수식/TAC 높이, 표/박스 내부 흐름, LINE_SEG 해석 중 어디에 있는지 분리한다.
2. 한컴오피스 기준처럼 조건 박스 내부 텍스트와 수식 줄이 겹치지 않도록 보정한다.
3. 기존 미주 간격 이슈(#1256, #1257)와 같은 원인인지, 별도 문단/수식 배치 문제인지 확인한다.
4. 가능하면 대상 페이지/문단에 대한 좌표 단언 테스트를 추가해 재발을 방지한다.

## 선행 사실

- 기준 문서: `samples/3-10월_교육_통합_2022.hwp`
- 기준 위치: 5쪽, `문28)` 조건 박스 내부
- 사용자 재현 화면: `http://localhost:7700`
- 이슈: [#1261](https://github.com/edwardkim/rhwp/issues/1261)
- 작업 브랜치: `local/task_m100_1261`
- 현재 `gh` 로그인 계정은 `jangster77`이며, 저장소 assignee 지정 권한이 없어 assignee 변경은 시도하지 않는다.

## 조사 범위

- `dump-pages`로 5쪽 문단/표/컨트롤 배치 목록을 확인한다.
- `dump`로 `문28)` 조건 박스 주변 문단의 ParaShape, LINE_SEG, 수식/TAC 컨트롤 속성을 확인한다.
- render tree 또는 SVG 좌표에서 조건 박스 내부 텍스트와 수식 bbox가 실제로 겹치는 지점을 측정한다.
- 기존 회귀 테스트 `tests/issue_1139_inline_picture_duplicate.rs`의 `3-10월_교육_통합_2022.hwp` 관련 가드와 충돌하지 않는지 확인한다.

## 위험

- `3-10월_교육_통합_2022.hwp`는 미주 흐름과 다단 배치 보정 이력이 많아, 단순 줄높이 증가는 후속 문항 위치나 페이지 수를 흔들 수 있다.
- 수식/TAC 높이를 공통으로 키우면 다른 시험지 문서의 줄 간격이 과도하게 벌어질 수 있다.
- 조건 박스가 표/글상자/문단 테두리 중 어느 구조인지에 따라 수정 위치가 달라진다. 원인을 확인하기 전에는 공통 렌더러에 넓은 보정을 넣지 않는다.

## 검증

- `cargo test --test issue_1139_inline_picture_duplicate -- --nocapture`
- 신규 회귀 테스트 추가 시 해당 테스트 단독 실행
- 필요 시 `cargo test --tests`
- Rust/WASM 변경 후 `wasm-pack build --target web --out-dir pkg`
- rhwp-studio에서 5쪽 `문28)` 조건 박스가 한컴 기준처럼 겹치지 않는지 작업지시자 시각 판정
