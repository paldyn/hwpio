## 1. 배경

표 셀 내부 그림/도형 우클릭 → "개체 속성(P)..." 클릭 시 다이얼로그 미표시. 콘솔:
```
[CommandDispatcher] 커맨드 실행 실패: insert:picture-props 렌더링 오류: 지정된 컨트롤이 Shape이 아닙니다
```

### 4-layer cellIdx 누락 (이슈 본문)

| Layer | 위치 | 문제 |
|-------|------|------|
| Handler | `rhwp-studio/src/command/commands/insert.ts:184` | `picturePropsDialog.open(sec, ppi, ci, type, headerFooter)` — `ref.cellIdx`, `ref.cellParaIdx` 전달 누락 |
| Dialog | `rhwp-studio/src/ui/picture-props-dialog.ts:219-225` | `open()` 시그니처에 cellIdx 매개변수 없음 |
| WASM API | `src/wasm_api.rs:2364, 2658` | `getPictureProperties` / `getShapeProperties` — cellIdx 매개변수 없음 |
| 호출 사이트 | (위 3 layer 의 호출자) | cellPath 전달 안 함 |

## 2. 접근 방향

기존 `_by_path` 패턴 (`get_cell_paragraph_count_by_path` 등, `cursor_nav.rs:552 resolve_cell_by_path`) 일관성:

- 신규 WASM API 4개:
  - `getCellPicturePropertiesByPath(sec, parent_para, cell_path_json, inner_control_idx)` + setter
  - `getCellShapePropertiesByPath(...)` + setter
- path JSON 스키마: `[{"controlIdx":N,"cellIdx":M,"cellParaIdx":P}, ...]` (마지막 segment 로 셀 도달, `inner_control_idx` 가 셀 paragraph 내 picture/shape control idx)
- nested table 자동 확장 가능 (path 배열 길이 늘림)
