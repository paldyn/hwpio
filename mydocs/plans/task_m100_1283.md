# Task #1283 수행 계획서

## 이슈

- GitHub Issue: #1283
- 제목: HWPX OLE chart Contents rendering missing for 143E433F503322BD33

## 배경

`samples/143E433F503322BD33.hwp`는 #1268/#1251 처리로 legacy OLE `/Contents` 차트가 SVG로 렌더링된다.
하지만 동일 계열 HWPX 변환본인 `samples/hwpx/143E433F503322BD33.hwpx`는 여전히 OLE placeholder 경로로 남는다.

## 원인 가설

HWPX 패키지에는 `BinData/ole3.ole`이 포함되어 있고 `Contents/section0.xml`의 `<hp:ole>`도
`binaryItemIDRef="ole3"`으로 참조한다. 그러나 `Contents/content.hpf`에서 해당 item이
`isEmbeded="0"`으로 기록되어 현재 파서가 외부 링크로 분류하고 ZIP 내부 bytes 로딩을 건너뛴다.

또한 HWPX 내부 OLE stream은 HWP5 Storage와 동일하게 `[4-byte size][CFB bytes]` prefix 계약을
가질 수 있으므로, 렌더러에는 prefix 제거 후 CFB bytes를 전달해야 한다.

## 처리 범위

1. HWPX `BinData/*.ole` 내부 패키지 항목은 `isEmbeded="0"`이어도 `BinDataType::Storage`로 분류한다.
2. 외부 절대경로 이미지 fallback (#873)은 유지한다.
3. 내부 OLE bytes 로딩 시 4-byte size prefix가 있으면 제거한다.
4. HWPX fixture 기반 회귀 테스트를 추가한다.
5. 기존 2단 차트 레이아웃 테스트는 placeholder와 실제 OLE chart SVG 모두 지원하도록 갱신한다.
6. HWPX→HWP 저장 시 OLE Storage `BIN_DATA` attr과 `SHAPE_COMPONENT_OLE` payload의
   `bin_data_id` 보존을 검증한다.
