# 오늘 할일 - 2026년 4월 23일

## Task #146 — text-align.hwp SVG ↔ 한컴 PDF 렌더링 차이 보정

### 배경

작업지시자가 제공한 `text-align.hwp` 1페이지 문서에 대해 `rhwp export-svg` 출력과 한컴 오피스 PDF 출력(`text-align.pdf`)을 150dpi로 비교한 결과, 페이지 레이아웃 단계부터 시각적 차이가 식별됨.

### 식별된 버그 후보 (3건)

| 후보 | 증상 | 성격 |
|------|------|------|
| ① | 제목 "□ 국어…" 에서 □ 뒤 공백 x-advance 누락 (□→국 간격 18.82px, 정상 ~31px) | 실제 버그 |
| ② | `Alignment::Justify` 가 레이아웃에만 저장되고 SVG 출력 x 좌표에 반영 안됨 → 오른쪽 여백까지 퍼지지 않음 | 실제 버그 |
| ③ | 음수 indent(hanging) 후속 줄 x 기준점이 PDF와 어긋남 | 실제 버그(검증 필요) |
| (참고) | HY헤드라인M fallback 으로 제목 굵기 약함 | 폰트 전략(별도 범위) |
