## 3. 기술 전략: Rust 네이티브

**WASM(WebView)이 아닌 Rust → iOS 네이티브 빌드**

| | 한컴독스 (WebView 하이브리드) | 알한글 (Rust 네이티브) |
|---|---|---|
| 렌더링 | 웹 엔진 | **Metal / Core Graphics** |
| 성능 | 브라우저 오버헤드 | **네이티브 최적화** |
| 파일 연동 | 제한적 | **iOS Files 앱 직접 통합** |
| 메모리 | JS GC | **Rust 직접 관리** |
| 스크롤 | 웹 스크롤 | **UIKit 네이티브 60fps** |
| 입력 | 키보드 문제 多 | **네이티브 키보드 + Pencil** |

빌드: `cargo build --target aarch64-apple-ios` + Swift UI FFI

## 4. 핵심 기능: 레이어 구조

### 파일 구조

```
┌─────────────────────────────┐
│  어노테이션 레이어           │  ← Apple Pencil 필기, 형광펜, 메모
├─────────────────────────────┤
│  편집 가능한 문서            │  ← 텍스트 수정, 표 편집
├─────────────────────────────┤
│  원본 HWP                   │
└─────────────────────────────┘
```

### .rhwp 파일 포맷

HWP 원본 + 어노테이션 레이어를 **하나의 파일**로 묶음.

### 내보내기

| 내보내기 | 대상 | 내용 |
|---------|------|------|
| .rhwp | 같은 앱 사용자 | HWP + 어노테이션 전체 보존 |
| .hwp | 선생님 (한컴 사용) | 텍스트 편집만 반영 |
| .pdf | 누구나 | HWP + 어노테이션 합쳐서 평탄화 |

## 5. 사용 시나리오

### 학생: 시험 문제 풀기

```
카카오톡에서 시험문제.hwp 받음
  → 탭 한 번으로 열기
  → 답안은 타이핑 (키보드)
  → 풀이 과정은 Apple Pencil로 필기
  → 저장 (.rhwp)
  → 선생님에게 .pdf로 전송
  → 나중에 다시 열어서 복습
```

### 선생님: 채점

```
학생이 보낸 시험문제.rhwp 열기
  → 학생 답안 (타이핑) + 풀이 과정 (필기) 확인
  → 빨간 펜으로 채점 + 코멘트
  → 학생에게 다시 전송
```

### 레이어 누적

| 레이어 | 작성자 | 색상 |
|--------|--------|------|
| 원본 시험 문제 | 선생님 | (문서) |
| 답안 타이핑 | 학생 | (문서 편집) |
| 풀이 과정 필기 | 학생 | 파란펜 |
| 채점 + 코멘트 | 선생님 | 빨간펜 |
| 오답 정리 | 학생 | 초록펜 |
