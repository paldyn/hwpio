# 외부 공개 문서 자기검열 체크리스트

## 규칙

rhwp 를 지켜보는 사람들이 늘고 있다 — oosmetrics Top 2 등재, 8명 외부 기여자, Chrome/Edge 스토어 승인, 특정 상용 제품 부사장의 기여 의사 등. 프로젝트는 이미 **혼자 만드는 것이 아니며**, 문서는 **혼자 읽는 것도 아니다**. 따라서 외부 노출 가능 문서를 작성·수정할 때 아래 체크리스트를 반드시 통과시켜야 한다.

**Why**: 관계자가 읽고 불쾌해하면 기여 의사가 철회되거나, 법적 리스크(비교광고·명예훼손·상표)로 번지거나, 반례 하나로 프로젝트 평판이 훼손될 수 있다. 2026-04-22 실제로 특정 상용 제품 관계자 기여 의사 접수 후 긴급 문서 정비(21 파일) 가 필요했다.

**How to apply**: 외부 공개 가능성이 있는 문서는 커밋 전 이 체크리스트로 점검. `mydocs/report/`, `mydocs/tech/`, `README*`, `CONTRIBUTING.md`, Discussions 포스트, 릴리즈 노트, 스토어 설명, 외부 공유용 슬라이드·블로그 모두 대상. 내부 전용(`mydocs/plans/`, `mydocs/working/`) 도 커뮤니티 투어 대상이 되면 동일 적용.

## 체크리스트

### 1. 구체 회사·제품명 언급

- [ ] 특정 회사·제품을 **비교 문맥**에서 거명하지 않았는가?
  - NG: "폴라리스오피스 대비 1/10 크기", "X 제품은 처리 실패"
  - OK: "일부 상용 제품 대비", "일부 상용 웹오피스"
- [ ] **긍정적 맥락**이어도 거명에 동의가 있는가? (없으면 일반화)
- [ ] **한글과컴퓨터** 는 예외적으로 거명 가능하나, 표현 수위는 중립 유지. "한컴 서버 의존" 같은 사실 서술은 OK, "한컴을 넘어선다" 같은 우월 주장은 단서 필요.

### 2. 순위·비율·최상급 주장

- [ ] "**세계 최초**", "**유일한**", "**전무**", "**최고**" 류 표현을 단서 없이 썼는가?
  - NG: "WebAssembly 기반 HWP 저장은 세계 최초"
  - OK: "우리가 확인한 범위 안에서는 세계 최초", "공개 구현 중 최초로 추정"
- [ ] 비율/수치 비교에 **측정 조건** 을 같이 썼는가?
  - NG: "렌더링 품질 95%"
  - OK: "KTX.hwp 1페이지 기준 자가 비교 95%. 문서·환경에 따라 차이 발생"
- [ ] "**X 수준 달성**" 류 주장은 측정 기준을 명시했는가?

### 3. 사용자 인용·인물 언급

- [ ] **개인 실명**, 이메일, 연락처가 노출되지 않았는가? (GitHub `@아이디` 는 공개 핸들이라 OK, 하지만 개인 정보는 금지)
- [ ] 인용문의 **수위** (욕설·비난) 가 프로젝트 톤에 맞는가? 원문 보존이 필요하면 **문서 앞에 "외부 사용자 발언 인용, rhwp 견해 아님" 주석** 을 반드시 둔다.
- [ ] GitHub 아이디로 언급된 사람의 **동의 범위**가 확실한가? (기여자 감사 목록은 OK, 개인 메시지 내용 공개는 NG)
- [ ] 비공개 대화(메일·DM·오프라인) 내용이 문서에 담겼는가? (담겼다면 공개 적합성 재검토)

### 4. 공공기관·조직 언급

- [ ] 공공기관·조직명이 **도입 사례처럼 읽히는 위험**이 있는가?
  - NG: "LG인화원, 경남교육청 등 B2B 웹오피스" (rhwp 채택이 아닌데 채택처럼 읽힐 수 있음)
  - OK: 거명 없이 "일부 공공/교육 기관"
- [ ] **후원/제휴 관계로 오해**될 표현이 있는가? (Trademark 섹션 스타일로 명시)

### 5. 기술 호환성·품질 주장

- [ ] "**100% 호환**", "**한컴 동일**" 류 주장이 **경계 케이스** 를 포함한 전수 검증인가? 아니라면 "샘플 N 건에서 호환 확인" 형식으로 완화.
- [ ] 타사 제품을 **처리 실패** 로 명시한 표는 꼭 필요한가? 최소한 측정 환경·버전·샘플 명시. 가능하면 일반화.
- [ ] **성능 우위** 주장에 재현 가능한 조건이 있는가?

### 6. 문서별 예외

- **`mydocs/report/x_hwp_viewer_voices.md`**: 외부 사용자 발언 원문 보존이 목적. 원문 수정 금지. 단 문서 **첫머리 주석** 필수 ("외부 공개 인터넷 발언 인용, rhwp 프로젝트 견해 아님").
- **archives (`mydocs/**/archives/`)**: 과거 타스크 기록이라 역사적 가치 존중. 단 외부 공개 가능성 있으면 본 체크리스트 동일 적용 (과거라고 면제되지 않는다).
- **feedback (`mydocs/feedback/`)**: 내부 소통용이나 리포지토리에 공개되어 있음. 민감 건은 `.gitignore` 또는 `/home/edward/.claude/...` 경로로 옮긴다.

### 7. 자동 발견 도구

```
# 회사·제품명 자기검열 grep (커밋 전 체크)
grep -rn "폴라리스\|Polaris\|polaris" mydocs/ README*
grep -rn "세계 최초\|유일한\|전무\|최고" mydocs/ README*
grep -rn "100% 호환\|한컴 동일\|한컴 수준" mydocs/ README*
```

AI (Claude) 는 문서 생성·수정 후 자동으로 이 grep 패턴을 돌려 자가 검증할 것.

## 이 체크리스트 자체 위반 시 기록

본 체크리스트 위반이 실제 문제로 번진 사례는 `mydocs/troubleshootings/` 에 기록하고 본 메모리도 보강한다. 2026-04-22 폴라리스오피스 건이 첫 사례 (mydocs/tech/ 등 21 파일 긴급 정비).
