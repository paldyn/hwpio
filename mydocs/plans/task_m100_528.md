# Task #528 수행계획서 — 옛한글 글리프 폴백 지원

**작성일**: 2026-05-02

## 1. 배경 / 증상

`samples/exam_kor.hwp` 페이지 17 의 중세국어 (고어) 표기가 미렌더된다. 대표 예:

```
'다'(되다)         ← 빈 자리 = 결합 자모 합자 미렌더
'(혼자)'
'​​'              ← 시각적 빈 공간
'스스​'
```

`rhwp dump-pages samples/exam_kor.hwp -p 16` 출력에서 다음 paragraph 들에 옛한글 결합 자모 시퀀스 확인:

- pi=4 "한자를 적을 때는, ..."
- pi=5 "한편 …'ㅸ'을 가진 '다'(되다), '(혼자)'"
- pi=9 "②-'오​​'(오늘)과 ..."
- pi=11 "③-현대 국어 '바닷물'의 ..."

## 2. 본질

### 2-1. 옛한글 인코딩

HWP 는 옛한글을 유니코드 결합 자모 시퀀스로 저장:

| 영역 | 유니코드 | 용도 |
|------|---------|------|
| Hangul Jamo | U+1100~11FF | 초성/중성/종성 (현대 + 일부 옛 자모) |
| Hangul Jamo Extended-A | U+A960~A97F | 옛 초성 (확장) |
| Hangul Jamo Extended-B | U+D7B0~D7FF | 옛 중성/종성 (확장) |
| Hangul Compatibility Jamo | U+3130~318F | 단일 자모 (합자 무관) |

### 2-2. 합자 메커니즘

폰트의 **OpenType GSUB / CCMP** 피처로 초성+중성(+종성) 시퀀스를 단일 음절 글리프로 합성한다. 이 피처가 없는 폰트는 자모를 개별 글리프로 나란히 배치하거나 빈 글리프로 표시한다.

### 2-3. 현 fallback 체인 한계

`src/renderer/style_resolver.rs:565~600` 의 fallback 체인:

```
한컴바탕 → 함초롬바탕 → (시스템 세리프)
```

`mydocs/tech/font_fallback_strategy.md:283~308` 의 웹 fallback:

```
@font-face Noto Serif KR
  unicode-range: U+1100-11FF, U+A960-A97F, U+D7B0-D7FF
```

문제:
- **Noto Serif KR (CJK KR weight 변형)** 는 Hangul Jamo 글리프를 포함하지만 합자 GSUB 피처는 미지원 (확인 필요 — 1단계 검증)
- **시스템 세리프** (AppleMyungjo 등) 는 옛한글 합자 미지원
- **함초롬바탕 LVT** 는 한컴 라이선스로 번들 불가

### 2-4. 코드 인프라 현황 (긍정적)

`src/renderer/style_resolver.rs:377-415` 의 `detect_lang_category` 가 이미 옛한글 영역 (U+1100-11FF, U+A960-A97F, U+D7B0-D7FF) 을 한국어로 인식한다. 즉 **언어 카테고리 분류는 정상**이며, 폰트 fallback 만 보강하면 된다.

또한 **Task #122** (2026-02-20 완료) 에서 옛한글 자모 클러스터 폭 계산은 이미 구현됨 (`mydocs/working/archives/task_122_report.md`). 즉 **레이아웃 계산은 정상**, 글리프 출력만 결함.

## 3. 해결 방향

### 3-1. 라이선스 안전 폰트 선정

| 후보 | 라이선스 | 합자 지원 | 파일 크기 | 비고 |
|------|---------|----------|----------|------|
| **본명조 (Source Han Serif K)** | SIL OFL 1.1 | ✅ v2.001+ | ~17 MB (full) | Adobe + Google, 가장 검증됨 |
| 나눔명조 옛한글 | SIL OFL 1.1 | ✅ | ~5 MB | 네이버, 한국 친숙 |
| KoPubWorld 바탕 옛한글 | "자유 사용" (OFL 미명시) | ✅ | ~12 MB | 라이선스 텍스트 모호성 |

**결정**: **본명조 (Source Han Serif Korean) Regular** — SIL OFL 1.1 명확성 + 검증 이력 + 함초롬바탕과 같은 명조 계열.

### 3-2. 번들 전략

| 영역 | 전략 |
|------|------|
| **WASM 웹 빌드** (`rhwp-studio/public/fonts/`, 브라우저 확장) | 옛한글 영역 **subset woff2** 번들 (~500KB~1MB). `@font-face unicode-range` lazy load |
| **네이티브 CLI** (`rhwp export-svg`) | `ttfs/FONTS.md` 정책 일관성 — 폰트 파일 Git 미포함. 사용자가 `--font-path` 로 지정 |
| **단위 테스트** | 시스템 폰트 의존 없이 메트릭스만 검증 |

### 3-3. fallback 체인 보강

`src/renderer/style_resolver.rs::resolve_font_substitution` 에서 한국어 + 옛한글 코드포인트 영역에 **본명조 (Source Han Serif K)** 를 fallback 으로 추가. 현 시스템 세리프 → 본명조로 우선순위 변경.

웹 빌드 CSS:
```css
@font-face {
  font-family: "Source Han Serif K Old Hangul";
  src: url("fonts/SourceHanSerifK-OldHangul-subset.woff2") format("woff2");
  font-display: swap;
  unicode-range: U+1100-11FF, U+A960-A97F, U+D7B0-D7FF;
}

/* font-substitution chain 말단에 추가 */
font-family: "한컴바탕", ..., "Noto Serif KR", "Source Han Serif K Old Hangul", serif;
```

## 5. 회귀 / 리스크 분석

### 5-1. 영향 범위 매트릭스

| 영역 | 영향 | 검증 방법 |

| 옛한글 합자 표시 | + (개선) | `exam_kor` p17 시각 판정 |
| 글자폭 / 페이지 수 | ? (검증 필요) | 7 샘플 페이지 수 변동 확인 |
| WASM 페이로드 | + ~500KB-1MB | 측정 |
| 네이티브 SVG export | 0 (변경 없음) | 회귀 테스트 |

### 5-2. 메모리 가이드 정합

| 메모리 | 적용 |
|--------|------|

| `feedback_essential_fix_regression_risk` | 본질 정정 회귀 위험 → 광범위 샘플 + 한컴 정답지 비교 |
| `feedback_rule_not_heuristic` | OpenType 합자는 폰트 표준 룰 → 휴리스틱 분기 추가 금지 |

### 5-3. 잔존 리스크

- **본명조 의 합자 커버리지 미완전 가능성** — 일부 옛한글 음절이 여전히 미렌더될 수 있음. Stage 1 에서 exam_kor 의 모든 옛한글 시퀀스를 추출하여 사전 검증
- **CSS unicode-range 분리 한계** — Canvas 2D 렌더링에서는 unicode-range 가 작동하지 않음. `font_fallback_strategy.md:321~339` 의 FontFace API 영역 검토 필요
- **나눔명조 옛한글 보완 가능성** — 본명조 미커버 영역이 있을 경우 보조 폰트 동시 번들 검토 (Stage 1 결과에 따라 결정)
