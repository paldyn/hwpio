## 1. 문제 정의

`samples/hwp3-sample16-hwp5.hwp` 의 paragraph height 측정이 HWP3 native 의 약 **2배**. 같은 11 paragraphs (pi=450~460) 가 HWP3 에서 fit, HWP5 변환본 에서 PartialParagraph 로 split + 외곽선 overflow.

### 1.1 정량 측정 (sample16 p23 = idx=22)

| pi | HWP3 h | HWP5 변환본 h | 배율 |
|----|--------|-------------|------|
| 450 | 59.5 | 118.5 | 2.0× |
| 451 | 75.0 | 116.6 | 1.6× |
| 452 | 57.7 | 88.9 | 1.5× |
| 453 | 40.3 | 61.1 | 1.5× |
| 454 | 75.0 | 116.6 | 1.6× |
| 455 | 40.3 | 61.1 | 1.5× |
| 460 | 111.5 | 173.9 | 1.6× |

→ HWP5 변환본 paragraph height **평균 1.5~2배** 큰 측정.

### 1.2 증상

- **p23 외곽선 overflow**: pi=460 PartialParagraph 로 split, 시각적 외곽선 below text overflow
- **p21 alignment 잔존**: PR #1036 fix 후에도 4 미정합 페이지 — paragraph height 차이가 누적 paginator break 위치 결정 영향

## 2. Root cause 가설

### 2.1 가설 A — font metric line_height 차이 (Task #1008 격차 D 연장)

Task #1008 격차 D 에서 폰트명 매핑 (신명조 → HY신명조) 으로 advance width 정합. 그러나 **line_height** 는 정합 안 됨:
- HWP3 char_shape → IR CharShape.base_size (× 4)
- HWP5 char_shape → IR CharShape.base_size
- line_height 계산 = `base_size × line_spacing_ratio`

base_size 동일 (1400 HU) 인데도 line_height 가 다른지 측정 필요.

### 2.2 가설 B — line_spacing_ratio 차이

ParaShape 의 `line_spacing_type` + `line_spacing` 값 산출 차이:
- HWP3: hunit 변환 + percent ratio
- HWP5: HWPUNIT 직접

값 자체는 같아도 적용 시 차이 가능.

### 2.3 가설 C — spacing_before/after 산출 차이

paragraph 의 spacing_before/after 가 HWP3 와 HWP5 변환본 사이 다르게 산출되어 누적.

### 2.4 가설 D — ParaShape 의 line height 정의 자체 차이

HWP3 와 HWP5 의 paragraph layout spec 자체가 다름. variant 한정 보정 필요.
