# 수행계획서 — Task #1343: 수식 조건부 막대 U+E04D(PUA) 두부(▦) 렌더 수정

- **이슈**: #1343 (M100)
- **브랜치**: `local/task1343` (base: `stream/devel`)
- **관련**: #1342(∩/∪ 확대)에서 분리된 후속 이슈

## 1. 증상

16페이지 문24 `P(A|B)`의 조건부 막대 `|`가 HWP 수식 스크립트에서 PUA 문자 `U+E04D`로
인코딩되어 있어, 렌더 시 글리프 미존재로 두부(▦) 박스로 표시된다.

- 샘플: `samples/3-09월_교육_통합_2024-구분선아래20구분선위20.hwpx` 16페이지
- 스크립트: `rm P LEFT ( it A  B RIGHT ) = ... over ... RIGHT .`

## 2. 원인 분석

수식 토크나이저(`src/renderer/equation/tokenizer.rs`)는 ASCII가 아닌 문자를
"기타 문자(한글 등)" 분기에서 그대로 `Text` 토큰으로 만든다(459행 부근). PUA 문자
`U+E04D`도 이 경로를 타서 원형 코드포인트가 보존되고, 폰트에 해당 글리프가 없으므로
두부(▦)로 렌더된다.

한컴 수식 편집기는 일부 기호를 PUA(U+E000~F8FF) 영역에 저장한다. `U+E04D`는 조건부
확률 막대 `|`에 해당한다. 토크나이저/심볼 매핑에 HWP 수식 PUA 매핑이 부재한 것이 원인.

- 샘플 전수 조사 결과 수식 스크립트에서 사용되는 PUA 문자는 `U+E04D` 1종뿐이다.

## 3. 수정 범위

| 파일 | 변경 |
|------|------|
| `src/renderer/equation/symbols.rs` | HWP 수식 PUA→표준기호 매핑 테이블 + 조회 함수 추가 |
| `src/renderer/equation/tokenizer.rs` | non-ASCII Text 분기 직전에 PUA 매핑 조회 추가 |

- 매핑이 없는 PUA 문자는 기존과 동일하게 `Text` 토큰으로 폴백한다(안전).
- 수식 도메인 한정 변경. 공통 렌더러/레이아웃에는 손대지 않는다.

## 4. 검증 방법

1. `cargo build` / `cargo test` 통과
2. `export-svg`로 16페이지 출력 → `P(A|B)` 막대가 두부 없이 `|`로 렌더되는지 확인
3. 기존 `|`(`vert`, 단일 `|`) 렌더와 동일 출력인지 확인
