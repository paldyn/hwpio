# Task #1285 수행 계획서

## 이슈

- GitHub Issue: #1285
- 제목: TAC 표 2개가 한 셀에 배치된 오른쪽 정렬 문단에서 inline 폭 계산 오류

## 배경

`samples/21_언어_기출_편집가능본.hwp` 1페이지 머리말 영역에서 `성명` 표와 `수험번호` 표가
한 셀의 같은 문단 안에 `treat_as_char` 표 2개로 배치되어 있다.

이 부모 문단은 오른쪽 정렬인데, rhwp는 한컴에디터보다 `성명`/`수험번호` 표 묶음을 왼쪽으로 배치한다.
PR #1277 검토 중 발견되었지만, #1277 적용 전 `origin/devel`에서도 동일 SVG가 생성되므로 기존 결함으로 분리했다.

## 재현 대상

- `samples/21_언어_기출_편집가능본.hwp`
- page 1
- 머리말 영역의 큰 표 내부
- `성명` / `수험번호` 영역

## 현재 관찰

- 부모 셀 문단: 오른쪽 정렬
- 같은 부모 셀 문단 내부: TAC 표 2개가 inline sequence로 배치됨
- `수험번호` 표 내부 첫 셀: 오른쪽 정렬 텍스트
- 현재 `수험번호` textRun 시작 x가 기대보다 약 8px 왼쪽에 위치

## 원인 가설

`layout_composed_paragraph` 또는 그 입력이 되는 composer 측에서 줄 정렬용 inline 폭을 계산할 때,
TAC 표 2개와 사이 공백을 하나의 연속 inline sequence로 합산하지 못하는 것으로 본다.

점검 후보:

1. `src/renderer/composer.rs`의 TAC table metrics 수집
2. `src/renderer/layout/paragraph_layout.rs`의 `layout_composed_paragraph`
3. `tac_offsets_px`, `line_tac_offsets`가 run 경계와 line 경계에서 동일한 폭 정보를 쓰는지 여부
4. 오른쪽 정렬 또는 justify 분기에서 텍스트 폭과 TAC 폭을 대체 관계로 처리하지 않는지 여부
5. 부모 셀 문단 안 multiple TAC table sequence의 total inline width 산출 방식

## 진행 계획

1. page 1을 디버그 오버레이 SVG와 render tree JSON으로 내보낸다.
2. `dump`로 머리말 영역 부모 셀 문단의 ParaShape, LINE_SEG, controls 순서를 확인한다.
3. `성명`/`수험번호` TAC 표 2개의 common width, inline offset, textRun bbox를 수치화한다.
4. 현재 `origin/devel` 기준 기대/현상을 고정할 회귀 테스트를 작성한다.
5. composer/layout 중 한 곳에서 multiple TAC sequence 폭 계산을 최소 수정한다.
6. SVG 시각 판정용 산출물을 만든다.
7. 관련 회귀 테스트와 포커스 테스트를 실행한다.
8. rhwp-studio 확인이 필요하면 WASM을 빌드한다.

```bash
docker compose --env-file .env.docker run --rm wasm
```

## 완료 기준

1. `성명`/`수험번호` TAC 표 묶음이 한컴에디터의 오른쪽 정렬 배치와 시각적으로 정합한다.
2. `수험번호` 표 내부 오른쪽 정렬 textRun 시작 x가 기대 위치로 이동한다.
3. multiple TAC table inline sequence가 포함된 문단에 대한 회귀 테스트가 추가된다.
4. 기존 TAC 표/inline picture/시험지 렌더링 회귀가 발생하지 않는다.
